# Project Overview
Web application which can run across all devices (desktop, laptop, tablets and mobile). The application will be used to fetch stock price, do advanced analysis to extract key insights with regards to stock performance and scrape the internet to retrieve news or information which could have influenced the stock prices in past or can influence the price in near future.

# Specifications
- Requirements: see 'docs/requirements.md'
- Technical specifications: see 'docs/tech-specs.md'

# Design Considerations
- Simplicity: This application is MVP. So keep the design simple. Do not overengineer the solution
- Security: do not expose security credentials, API keys etc
- Error handling: handle errors gracefully with user friendly message

# Architecture
- **Frontend**: React 19 + Vite 8 + Tailwind CSS v4 + Recharts + React Router v6 + Axios — plain JavaScript (.jsx), no TypeScript
- **Backend**: Python FastAPI + SQLAlchemy ORM, runs on http://localhost:8000
- **Database**: SQLite locally (`stock_analyst.db`); PostgreSQL in production (set `DATABASE_URL` env var)
- **Stock data**: yfinance (Yahoo Finance), incremental fetch — only fetches new data since last stored date
- **News**: RSS feeds via feedparser (Business Standard, LiveMint, Economic Times)
- **Auth**: JWT via python-jose, SHA256 password hashing

# Deployment (Render)
Platform: render.com — frontend (Static Site) + backend (Web Service) + PostgreSQL, all free tier.

## Steps to deploy
1. Push code to a GitHub repository
2. Go to render.com → New → Blueprint → connect the GitHub repo
3. Render reads `render.yaml` and creates all three services automatically
4. After first deploy, verify test users were seeded (seed_users.py runs in buildCommand)

## Environment variables set by render.yaml
| Service | Variable | Value |
|---|---|---|
| Backend | `DATABASE_URL` | Auto-linked from Render PostgreSQL |
| Backend | `JWT_SECRET` | Auto-generated by Render |
| Backend | `CORS_ORIGINS` | `https://stock-analyst-ui.onrender.com` |
| Frontend | `VITE_API_URL` | `https://stock-analyst-api.onrender.com` |

**Note**: If Render service names differ (name already taken), update the URLs in `render.yaml` to match.

## Local dev vs production
| Setting | Local | Production |
|---|---|---|
| Database | SQLite (`stock_analyst.db`) | PostgreSQL (via `DATABASE_URL`) |
| Backend URL | `http://localhost:8000` | `https://stock-analyst-api.onrender.com` |
| CORS | `*` (open) | Frontend URL only |

# Commands

## Backend
```bash
cd backend
pip install -r requirements.txt
cp .env.example .env          # set a strong JWT_SECRET in .env before running
python seed_users.py          # one-time: creates test users (user1@test.com / password1, etc.)
python main.py                # start FastAPI server on http://localhost:8000
# OR with hot-reload during development:
uvicorn main:app --reload
```

## Frontend
```bash
cd frontend
npm install
npm run dev        # start dev server on http://localhost:5173
npm run build      # Vite production build → dist/
npm run preview    # preview production build locally
npm run lint       # ESLint check (project is plain JS — no TypeScript)
```

# Known Gaps vs. Requirements
These features are in requirements.md but not yet implemented:

- **User registration**: No self-registration UI or endpoint. Uses pre-seeded test accounts only (seed_users.py)
- **Phone login**: Requirements allow email or phone; only email login is implemented
- **Automatic scheduling**: Requirements mention auto-triggering analysis on a schedule; only manual trigger exists
- **Financial Express news**: requirements.md lists it as a source but it is missing from RSS_FEEDS in news_service.py
